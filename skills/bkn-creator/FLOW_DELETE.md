# 删除流程（Delete）

## 流程目标

安全执行删除操作（整网或局部对象），并在删除前充分暴露影响面。

## 验证阶段：流程路由确认（进入删除流程前）

- MUST：先回显"识别结果 + 路由目标 + 输入摘要"，并发起确认请求
- MUST：仅当用户明确确认"进入删除流程"后，才可进入后续阶段
- Never：在验证阶段未确认前输出 `discover/preview/confirm` 细化内容或执行任何删除动作

## 渐进式阶段

1. `discover`：定位删除目标（`kn_id` / 对象类 / 关系类）
2. `preview`：展示删除清单与级联影响
3. `confirm`：等待用户明确"确认删除"
4. `execute`：委托子技能执行删除
5. `verify`：复查删除结果与残留引用
6. `report`：输出删除回执与后续建议

> 统一门禁规则见 `./COMMON_RULES.md`。对用户可见输出必须使用"用户回显模板（统一格式）"。

进入条件（前置）：验证阶段路由确认已通过。

## 展示策略（MUST）

- 默认仅输出摘要回显（删除目标、级联影响、风险、下一步）
- 同一轮仅允许一种主展示格式，禁止同一数据同时用表格与 YAML/JSON 重复展示
- 仅当用户明确请求时才输出详情（如"展开详情""导出 YAML""给我表格"）
- 详情输出一次仅允许一种格式（表格 或 YAML/JSON）

## 删除前必须展示（MUST）

- 删除目标清单（名称 + ID）
- 影响面：
  - 将失效的关系类
  - 可能成为孤悬的对象类
  - 可能影响的 Action / 绑定视图
- 风险提示：`不可恢复`

## 删除类型

- 整网删除：删除知识网络本体
- 局部删除：删除对象类/关系类并同步处理引用关系

## 核验清单

1. 目标已不存在（或标记为删除成功）
2. 无残留无效引用
3. 若局部删除，输出当前网络拓扑差异摘要

## 失败处理

- 删除失败时必须返回失败原因与重试建议
- Never 把含糊回复判定为"确认删除"

## 回显模板（对用户）

### 模板：流程路由确认（验证阶段）

```text
### 流程路由确认（验证阶段 | 请确认）
说明：
- 当前识别：这是"删除知识网络"请求
- 将进入流程：删除流程（FLOW_DELETE）
- 你提供的信息摘要：{input_summary}
- 说明：这一步只确认流程方向，还不会开始删除
下一步：你可以回复"确认进入删除流程"，或先调整删除目标后再继续。
```

### 模板：删除影响确认

```text
### 删除影响确认（删除流程 | 请确认）
说明：
- 删除目标：{target_list}
- 级联影响：{cascade_impact}
- 风险级别：不可恢复
下一步：请明确回复"确认删除"后继续执行。
```

```text
### 删除执行结果（删除流程 | 结果）
说明：
- 删除结果：{deleted_items}
- 残留引用检查：{residual_reference_check}
- 当前拓扑摘要：{topology_diff_summary}
下一步：如需恢复，请根据备份或重建方案处理。若需详情可回复：展开删除详情 / 导出 YAML。
```
