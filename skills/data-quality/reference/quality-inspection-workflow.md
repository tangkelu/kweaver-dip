# 质量检测流程

> **共享约束参考**: [核心约束](./core-constraints.md)

## 数据质量维度

> **详细说明**: [核心约束 - 维度类型约束](./core-constraints.md)

| 维度 | 说明 | 典型规则类型 |
|------|------|-------------|
| **完整性** | 数据是否存在缺失值 | 空值检查、必填项检查 |
| **及时性** | 数据是否按时更新 | 报告中可能展示，默认不作为自动补规则目标 |
| **规范性** | 数据格式是否符合标准 | 格式检查、编码规则检查 |
| **唯一性** | 数据是否重复 | 重复值检查、主键唯一性检查 |
| **准确性** | 数据值是否正确 | 范围检查、逻辑校验 |
| **一致性** | 数据在不同系统间是否一致 | 报告中可能展示，默认不作为自动补规则目标 |

## 详细流程说明

### 阶段1: 用户意图识别与触发

#### 1.1 识别质量检测意图

| 触发关键词 | 说明 |
|-----------|------|
| "质量检测" | 明确的质量检测请求 |
| "执行检测" | 执行质量检测操作 |
| "开始质检" | 启动质量检测流程 |
| "检测数据质量" | 数据质量检测请求 |
| "运行质量规则" | 运行已配置的质量规则 |

### 阶段2: 视图质量规则配置检查

#### 2.1 获取相关视图列表

**场景A: 知识网络场景**
```
通过知识网络对象类获取视图
    ↓
获取对象类列表
    ↓
提取对象类.data_source.id（统一视图ID）
    ↓
构建视图ID列表
```

**场景B: 数据源场景**
```
获取数据源下的所有视图
    ↓
GET /api/data-view/v1/form-view?datasource_id={id}
    ↓
提取视图ID列表
```

**场景C: 直接指定视图**
```
使用用户指定的视图ID列表
```

#### 2.2 扫描每个视图的质量规则

```http
GET /api/data-view/v1/explore-rule?form_view_id={视图ID}
```

检查项:
- 是否配置了接口支持的基础维度规则
- 各维度规则数量
- 规则启用状态
- 是否有至少一个启用的规则

#### 2.3 视图分类

> **完整分类标准**: [核心约束 - 视图分类标准](./core-constraints.md)

| 分类 | 条件 | 说明 | 处理 |
|------|------|------|------|
| ✅ 正常 | 有报告 + 有规则 | 规则配置完整 | 跳过，无需自动配置 |
| ⚠️ 待配置 | 有报告 + 无规则 | 规则配置不完整 | 进入阶段3自动配置 |
| 🔄 待检测 | 无报告 + 有规则 | 可直接检测 | 跳过配置 |
| ❓ 待配置+检测 | 无报告 + 无规则 | 需配置后检测 | 进入阶段3自动配置 |
| ⏭️ 已跳过 | 已删除/无权限 | 无法检测 | 记录，进入阶段4降级 |

#### 2.4 问题视图识别输出

```markdown
## 视图质量规则配置检查结果

**检查时间**: 2024-01-15 14:30:25
**检查范围**: 5个视图

### 检查结果摘要

| 类别 | 视图数量 | 处理方式 |
|------|----------|----------|
| ✅ 正常 | 2个 | 跳过 |
| ⚠️ 待配置 | 1个 | 自动配置 |
| ❓ 待配置+检测 | 1个 | 自动配置 |
| ⏭️ 已跳过 | 1个 | 降级处理 |
```

### 阶段3: 基础质量维度规则自动配置

#### 3.1 规则配置策略

```
对每个问题视图执行自动配置:
    │
    ▼
获取视图字段列表 → POST /api/data-view/v1/logic-view/field/multi
    │
    ▼
分析字段特征，推断适合的维度规则
    │
    ▼
创建质量规则 → POST /api/data-view/v1/explore-rule
```

#### 3.2 字段维度推断规则

> **详细推断标准**: [核心约束 - 自动规则推断](./core-constraints.md)

| 字段特征 | 推断维度 | 规则类型 | 示例 |
|----------|----------|----------|------|
| `primary_key=true` | 唯一性 | 主键唯一性检查 | ID字段 |
| `is_nullable=NO` | 完整性 | 非空检查 | 必填字段 |
| `standard_code` 不为空 | 规范性 | 标准编码检查 | 标准字段 |
| `data_type=INT/DECIMAL` | 准确性 | 范围检查 | 金额字段 |
| `data_type=DATETIME` + 时间戳特征 | 完整性 | 非空检查 | 创建时间字段 |
| 外键关联字段 | 完整性 | 非空检查 | 外键字段 |

#### 3.3 配置变更记录

## API调用顺序

| 序号 | API端点 | 方法 | 用途 |
|------|---------|------|------|
| 1 | `/api/eacp/v1/user/get` | GET | 配置验证 |
| 2 | `/api/data-view/v1/form-view` | GET | 获取视图列表 |
| 3 | `/api/data-view/v1/explore-rule` | GET | 扫描视图规则 |
| 4 | `/api/data-view/v1/logic-view/field/multi` | POST | 获取字段列表 |
| 5 | `/api/data-view/v1/explore-rule/repeat` | GET | 规则重名校验 |
| 6 | `/api/data-view/v1/explore-rule` | POST | 创建质量规则 |
| 7 | `/api/task-center/v1/work-order` | POST | 创建质量检测工单 |
| 8 | `/api/data-view/v1/explore-task` | GET | 跟踪探查任务状态 |

## 异常处理

| 异常场景 | 处理策略 | 用户反馈 |
|----------|----------|----------|
| 所有视图均无规则可配置 | 终止流程，提示检查数据源 | "该数据源下所有视图均无法配置质量规则" |
| 规则配置失败 | 记录错误，继续配置其他视图 | "视图xxx规则配置失败，已跳过" |
| 工单创建失败 | 记录错误，提示重试 | "工单创建失败，请稍后重试" |
| 探查任务全部失败 | 提示检查数据源连接 | "所有探查任务失败，请检查数据源连接" |
