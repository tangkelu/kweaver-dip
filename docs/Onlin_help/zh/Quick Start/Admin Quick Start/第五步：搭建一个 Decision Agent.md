在 KWeaver DIP 中，你可以通过创建 Decision Agent，将模型能力、业务知识与数据能力进行整合，用于支撑业务问答、任务决策、知识检索与流程执行等场景。
本文将带你完成 Decision Agent 的创建与基础配置，帮助你快速搭建一个可运行的 Agent。
## 功能说明
完成 Decision Agent 搭建后，你可以在系统中构建具备业务理解与执行能力的智能体，用于支持多种业务场景。
你可以实现以下操作：
• 创建 Decision Agent
• 配置模型与能力
• 绑定业务知识网络与数据源
• 进行调试与验证
• 发布并对外提供服务
## 前提条件
在开始前，请确保已满足以下条件：
• 已完成模型接入与配置
• 已构建业务知识网络
• 已准备好可用的数据源（如知识网络、文档库等）
• 已具备 Decision Agent 相关功能权限

## 使用说明
参考以下步骤，完成 Decision Agent 的创建与配置。通常包括 Agent 创建、能力配置、调试验证及发布等内容。
你可以根据实际业务需要，灵活完成相关配置。
## 步骤1：创建 Decision Agent
1. 进入【全局业务知识网络】→【Desicion Agent】，进入**开发** > **Decision Agent** 页面。
1. 单击**新建**按钮。
1. 进入 Decision Agent 的创建与配置页面。
1. 在基础信息区域，填写以下内容：
  - **名称**：输入 Agent 名称。
  - **简介**：简要描述 Agent 的用途与功能。
  - **头像**：上传或生成 Agent 头像。
  - **所属产品**：选择 Agent 对应的产品类型，例如 DIP。
完成基础信息后，即可进入后续配置流程。
## 步骤2：配置角色指令
角色指令定义了 Decision Agent 的身份、任务目标与执行边界，是影响 Agent 输出质量的关键配置项。
平台支持以下两种配置模式：
- **自然语言模式**：通过提示词描述 Agent 的角色、任务和约束，适合大多数场景快速搭建。
- **Dolphin（专家模式）**：通过编写 Dolphin 代码实现更复杂的任务逻辑和决策流程，适合高级配置场景。
建议在角色指令中明确以下内容：
- **角色定位**：说明 Agent 是谁，负责什么问题。
- **目标任务**：定义 Agent 需要完成的核心任务。
- **执行约束**：限定 Agent 的权限、行为边界、输出要求与异常处理方式。
如果你的 Agent 面向具体业务场景，建议在角色指令中尽量使用业务术语、业务规则与真实任务目标描述，以便模型更准确地理解任务。
## 步骤3：配置知识来源
为了让 Decision Agent 能基于业务知识进行分析与决策，你需要为其配置合适的知识来源。
支持的知识来源包括：
- **业务知识网络**：适用于承载企业业务概念、关系与规则，是推荐优先接入的知识来源。
- **知识条目**：适用于补充特定业务说明、文档知识或结构化知识内容。
如果你的 Decision Agent 用于企业业务决策、问答分析、流程辅助等场景，建议优先接入**业务知识网络**。相比仅依赖通用模型，接入业务知识网络后的 Agent 通常能够获得更准确的业务理解能力、更稳定的回答质量，以及更强的规则约束能力。
## 步骤4：配置技能与决策模型
根据业务需求，你可以为 Decision Agent 配置一个或多个技能模块，以扩展其能力边界。例如：
- 数据分析
- 合同匹配
- 检索增强
- 任务规划
同时，你还需要为 Agent 选择执行任务时所使用的模型。常见配置包括：
- **大语言模型**：适用于通用问答、内容生成与基础推理。
- **推理模型**：适用于结构化判断、复杂逻辑分析与多步骤决策。
通常情况下：
- 如果场景以知识问答、对话交互为主，可优先选择通用大语言模型；
- 如果场景以规则判断、任务分解、复杂决策为主，建议选择更适合推理任务的模型。
模型选择是否合适，将直接影响 Agent 的理解能力、推理深度与执行稳定性。
## 步骤5：完成高级配置
根据实际需要，你还可以继续完善以下配置项：
- **长期记忆**：保留历史交互信息，提升上下文连续性与个性化能力。
- **默认开场**：设置 Agent 启动时的问候语。
- **相关问题**：配置 Agent 可回答的问题方向。
- **任务规划**：设置任务拆解与执行规划策略。
- **预设问题**：提前配置常见问题，方便用户快速发起对话。
如果 Agent 涉及复杂业务任务，建议结合任务规划能力与长期记忆能力进行配置；但在启用长期记忆时，也需要同步关注数据隐私与合规要求。
## 步骤6：调试与验证
完成配置后，建议先在测试环境中进行调试，重点验证以下内容：
- 角色指令是否清晰、稳定；
- 模型是否已正确接入并可调用；
- 业务知识网络是否生效；
- 技能调用是否符合预期；
- 输出结果是否符合业务规则与场景要求。
如果出现回答不准确、无法调用知识、决策结果不稳定等问题，建议优先检查以下内容：
- 模型是否已正确配置；
- 角色指令是否表达明确；
- 知识来源是否已正确绑定；
- 技能是否已启用且与当前产品类型兼容。
## 步骤7：发布 Decision Agent
完成调试后，即可发布 Agent。
1. 单击**保存**，可将当前配置保存为草稿。
1. 单击**发布**，将当前版本正式部署到平台。
发布后，Decision Agent 即可供用户使用。后续如果需要修改配置，也可以在原有基础上继续编辑并重新发布，新版本会覆盖线上版本。历史版本会保留在配置信息页面，便于后续回溯与版本管理。
## 推荐配置顺序
建议按照以下顺序完成 Decision Agent 搭建：
1. 创建 Agent
1. 配置基础信息
1. 配置模型
1. 绑定业务知识与数据
1. 调试验证
1. 发布
## 注意事项
-  Decision Agent 的效果依赖模型能力与业务知识网络质量
- 建议优先完成业务知识网络构建后再进行 Agent 配置
- 模型参数设置会影响结果稳定性与响应质量
- 数据与知识配置需保持一致，避免出现错误调用
## 常见问题
**为什么 Decision Agent 无法正常运行？**
可能原因包括未配置模型、未绑定业务知识网络或数据源不可用。
**模型配置会影响什么？**
会影响理解能力、生成质量、响应速度及稳定性。
**是否必须绑定业务知识网络？**
不是必须，但绑定后可以显著提升业务理解能力与结果准确性。
**发布后可以修改吗？**
可以，但修改后建议重新验证，以确保结果符合预期。
