# 通用知识网络（GKN）构建流程
本文说明如何在具备「业务知识网络」能力的环境中，通过导入 GKN 模板、绑定数据视图并执行全量构建，完成通用知识网络的落地。
## 通用知识网络的作用
通用知识网络（GKN） 面向数据语义理解：在检索与问答等场景中，按语义关联并定位业务对象，进而映射到具体的逻辑视图（及字段等），使用户或下游能力能够「找对对象、落到视图、拿到数据」。在此基础上，领域知识网络在绑定真实业务数据时，可复用 GKN 已沉淀的语义对象与视图关系，减少重复配置并保证绑定方式一致。

### 小模型与 ConfigMap
知识网络构建会调用小模型能力。除在平台中完成小模型注册并保证地址可达外，还需在集群中为对应服务修改其所挂载的 ConfigMap（小模型开关、模型名称等均写在这些 ConfigMap 里，而非仅在界面点选）：
- `bkn-backend-cm`：`bkn-backend` 服务使用的 ConfigMap；将小模型相关开关设为启用（例如 `defaultSmallModelEnabled: true`，以清单中实际键名为准）。
- `ontology-query-cm`：`ontology-query` 服务使用的 ConfigMap；同样将小模型相关开关设为 `true`（键名与 YAML 片段以环境为准）。
上述 ConfigMap 中配置的小模型名称，须与 Deploy（部署清单 / 部署工作台中的模型部署配置） 里添加的小模型名称完全一致（含大小写与后缀习惯），否则构建或检索阶段可能找不到模型。
### 获取 model_id（小模型 ID）
模板里的 `model_id` 表示小模型 ID：须与构建所依赖的小模型一致。
1. 打开 部署工作台，进入 模型管理（与原先获取模型信息的入口相同）。
1. 切换到「小模型」列表（小模型 Tab / 子页），打开或刷新列表，使页面拉取小模型数据。
1. 使用浏览器 开发者工具（F12）→ 网络（Network） 面板抓包，选中小模型列表对应的接口请求，在响应 JSON 中查看。
1. 在响应体中找到 `data.model_id`（若 `data` 为数组，则在各列表项或分页结构中逐项对照所需小模型），将对应取值写入 通用知识网络（GKN）.json 的 `model_id` 字段。
## 操作步骤
### 步骤 1：导入 GKN 模板
在工作区目录中找到 通用知识网络（GKN）.json，在 业务知识网络 模块中执行导入。
### 步骤 2：配置数据连接并扫描
1. 在 数据连接 中新增数据源，选择系统内置库 af_main。
1. 对该数据源执行 扫描，确保库表与视图已同步到元数据。
### 步骤 3：打开 GKN 并进入对象类
返回 业务知识网络，打开已导入的 通用知识网络（GKN），进入 对象类列表。
### 步骤 4：为各对象类绑定 af_main 视图

### 步骤 5：触发全量构建
完成上述绑定后，进入 任务管理 → 新建 → 全量构建，执行构建任务。
## 注意事项
- 绑定顺序：步骤 4 中所有对象类的视图绑定须全部完成后再执行全量构建，否则可能出现语义对象不一致或未覆盖。
- 扫描问题：若扫描后仍看不到预期视图，请核对 af_main 的连接参数与扫描范围是否正确。
